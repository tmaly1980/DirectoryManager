Directory Manager 2.0 Documentation: Cross-Referenced Data

Adding and editing entries can involve simultaneously altering related records, such as the auxiliary groups or the mailing lists of a user. This provides the ability to alter an entry and all the related ones from one page.

Cross-referencing can be done in two ways. It can be enabled to allow you to alter these related entries, where you will have a list of values. You can add and remove values by clicking on 'Add' or 'Remove'. Having the related entries inline has a significant performance impact, due to extra database lookups.

* Note: These cross-referenced entries must already exist. I.e., you cannot add a new group to a user (via the user entry) when the group does not already exist.

To improve performance, it is possible to manage cross referenced entries without having a complete list of current values displayed.

This allows free-form entry of what to remove and what to add. In the case that you remove an entry that does not link to this entry, it will silently skip the operation. This scenario allows you to do modifications to the cross-referenced entries, without the poor performance of having to do extra database lookups prior.
