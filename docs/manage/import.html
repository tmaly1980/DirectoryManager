Directory Manager 2.0 Documentation: Bulk Import

Migrating data into LDAP via Directory Manager is simple, with the 'Bulk Import' feature. It is available TO ADMINISTRATORS ONLY via the 'Import' navigational button on the main menu (for the specific tree and respective module).

Import Link:
Bulk Import Page:

From this page, you can specify a pre-defined file format (if applicable for the module), such as /etc/passwd or /etc/shadow for the 'Users' module.

You can also specify that the input file is in LDIF format, or is a tab-separated text file (i.e. XLS) with or without the attribute names (on the first line). If a text file, you can specify a custom delimiter, such as ':', tab, comma, etc. You can also specify a filter (in the form of a boolean expression) so certain entries do not get imported (such as " '' > 100 " to ensure the 'root' user from /etc/passwd does not get imported).

You can specify a 'Mode', saying whether to simply add entries or to simply modify existing entries. You can also say to stop when an error occurs or to just keep on going. The output provided will display any errors if they occur. You can also specify an 'entry type', which ignores columns not meant for that entry type.

Finally, you specify which file to upload at the bottom.

You can specify that you want to 'Preview' the import or 'Generate LDIF' for it, specifying what exact LDIF-formatted entries will be entered into the system, without actually doing such. The 'Import' button does an actual insertion into the database with the status (result page) displayed in LDIF-format.
