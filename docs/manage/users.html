User entries can be one of multiple entry types, which can be changed at any time by an administrator:

Entry Types
  • Address Book Entry: Contains general information, contact information, employment information, and password (for logging in to DM as that user).
  • UNIX Entry: Contains all the information of an address book entry, as well as UNIX user account information, and mailing list information, suitable for use as UNIX accounts (as integrated with NSS/PAM of a UNIX system).
  • Samba 2 Entry: Contains Address Book and UNIX information, as well as account information for use with a Samba 2 server. Account information will, as much as possible, automatically determine samba schema attributes from the other related attributes.
  • Samba 3 Entry: Contains Address Book/UNIX information as well as account information for use with a Samba 3 server.
  • Samba 3 + Samba 2 Compat: Like a Samba 3 entry, but also includes the Samba 2 schema so an entry can function with either a Samba 3 OR Samba 2 server.
Search Fields

You can search for a user by any of the following fields:

  • Full Name
  • First Name
  • Last Name
  • Email
  • Username
  • Department
  • Company
  • Location
Sections

The following sections document each field available for changing (as well as their possible mandatory specification):

  • General Information:
    • Full Name (Required)
    • Photo: A JPEG or GIF photo can be uploaded into LDAP, and will be displayed on the view/edit page, when available.
    • Email: A list of emails that a user has. This can contain both emails for local organization's email server, aliases, as well as other emails that the user is generally available by.
    • Web Page: A list of web pages the user has available for viewing.
  • Contact Information:
    • Work Address
    • Location: Where this person works, is, etc...
    • Phone Number
    • Mobile (Cell) Phone Number
    • Fax Number
    • Pager Number
  • Employment Information:
    • Company
    • Department
    • Title
    • Manager: Displays in the form of an LDAP DN, but can be entered also in the form of a full name or username
  • UNIX Account Information (if a UNIX/Samba entry):
    • Username: i.e., their login name (Required)
    • User ID# (Required): An 'Auto-Generate' button is provided to generate a new value based upon what is available.
    • Primary Group (Required): Either the Group ID# or group name
    • Other Groups: Auxiliary groups for a user. Depending on cross-referencing settings, this may or may not display a current list
    • UNIX Home Directory (Required): Specifying the 'Create Home Directory' checkbox (which need not be checked) will ensure that the home directory exists on the (home directory) server. See Home Directory Overview for more details.
    • UNIX Shell: Can enter in /bin/false, etc to disable logging in
  • Mailing Lists (if a UNIX/Samba entry):
    • Depending on cross-referencing settings, this may or may not show a list of current lists.
  • Windows (Samba) Account Information (if a Samba 3/2 entry):
    • Windows Account Flags
    • Allowed Workstations: An empty list means no restrictions
    • Windows Home Drive
    • Windows Home Path
    • Windows Profile Path
    • Windows Logon Script
  • Password:
    • Password: Must be entered in twice. When switching to a Samba user from an Address Book/UNIX user, the password MUST be re-entered (or reset) to enable logging into Windows. The password can be randomly generated by the 'Randomly Generate' checkbox, in which case, the password will be displayed after saving to the directory.
    • Lock/Disable Account: Denies logins by putting a '!' in front of the UNIX/Samba passwords.
    • Macros

    The following macros are available within the 'User' module:

    • ${uidNum}: The next available user ID# in the system
    • ${FN}: First name, all letters capitalized
    • ${fn}: First name, all letters lowercased
    • ${Fn}: First name, first letter capitalized, other letters lowercased
    • ${Fi}: First (name) initial, capitalized
    • ${fi}: First (name) initial, lowercased
    • ${Ln}: Last name, first letter capitalized, other letters lowercased
    • ${LN}: Last name, all letters capitalized
    • ${ln}: Last name, all letters lowercased
    • ${li}: Last (name) initial, lowercased
    • ${Li}: Last (name) initial, capitalized
    • ${company_domain}: The email domain of the user's company, as defined in the domain map within the administrative menu
    Bulk Import

    The User Bulk Import page has the following pre-defined file formats:

    • /etc/passwd (without password, i.e. 'x' placeholder)
    • /etc/passwd (with password)
    • /etc/shadow
    • /etc/smbpasswd: Note: Entries must exist prior (i.e. from /etc/passwd)
    • You can, of course, specify raw LDIF, or a text file given a specified delimiter and whether or not columns are in the file or given via the form.
