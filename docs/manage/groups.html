Entry Types
 Group entries can be specified as one of the following entry types:
  • UNIX Group
  • UNIX + Samba 3 Group
Search Fields
 You can search for a group by any of the following fields (attributes):
  • Group Name
  • Group Member
Sections
 The following sections document each field available for changing (as well as their possible mandatory specification):
  • General Information:
    • Group Name (Required)
    • Description
    • Group ID# (Required): Can be autogenerated via the 'Auto-Generate' checkbox.
  • Members:
    • Lists UNIX users in this group (i.e. users who have this group as their AUXILIARY group).
    • You can click on 'View' after a member name to open a popup window displaying THEIR account information
Edit Group:

Depending on configuration settings for this tree, groups with a specific name can be tied to the well-known Samba groups. This causes the group to generate the proper Samba SID that represents the proper group, i.e. 'Domain Admins'. See the section on Samba Settings for more details on how to configure.

Macros

The following macros are available within the 'Group' module:

  • ${gidNum}: Generates the next available group number
Bulk Import

The Group Bulk Import page has the following pre-defined file format:

  • /etc/group
  • You can, of course, specify raw LDIF, or a text file given a specified delimiter and whether or not columns are in the file or given via the form.
