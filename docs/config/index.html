Directory Manager 2.0 Documentation: Configuration

Prior to use, Directory Manager needs to be configured. To do so,
  1. Accessing the Administration Site
  2. Interface Overview
  3. Adding the first Directory Tree
  4. Tree Settings Menu:

    This menu contains basic settings required for a tree to function within Directory Manager. It contains the following options:

    • Tree Name
    • Tree Abbreviation
    • Server Host
    • Base DN
    • Root DN
    • Access Control Profiles
    • Tree Initialization
  5. General Menu:

    The 'General' Menu lets you specify settings general to Directory Manager, not specific to a module.

    • Default Tree
    • Require Login
    • Implicit Edit
    • Implicit Link
    • Require Login
    • Alphabetical List Results
    • Entries Per Page
    • Request Recipients
    • Request Sender
    • Domain Map
  6. SSL/TLS Menu:

    The SSL/TLS Menu enables you to configure your SSL/TLS settings with Directory Manager to provide secure transmissions of both searches and modifications with your OpenLDAP server(s).

    • Encryption
    • SSL Peer Verification
    • Certificate Authority (CA) Directory
    • Certificate Authority (CA) File
    • Client SSL Certificate File
    • Client SSL Certificate Private Key File
  7. Modules Menu:

    The Modules Menu provides a way to configure settings for each module, such as the 'Users' or 'Groups' module. Each module may have it's own specific settings. The settings that can apply to any module include:

    • Access Level Requirements
    • Field Requirements
    • Customizable Form Drop-Downs

    Each module also may contain settings specific for the module. They include:

    • Users Module
    • Groups Module
  8. Samba 3 Menu:

    The 'Samba 3' Menu provides a way to designate settings required for Samba 3 to interoperate with Directory Manager. Designating the groups/users related to Samba 3 permit Directory Manager to generate the proper well-known RID for each entry. This menu includes the following settings:

    • Samba SID
    • Samba Domain Administrator
    • Samba Domain Administrator Group
    • Samba Domain Users Group
    • Samba Domain Guest Group
    • Samba Domain Guest User
  9. Access Control Menu:

    Access Control Profiles define the groups/teams that individuals belong to for administration of LDAP servers/trees. Profiles are separated from having a 1:1 relationship with servers so as to permit a single group being defined once, and used to administer several sets of LDAP servers/trees. One or more Access Control Profiles are then assigned to each specific directory tree.

    The Access Control Menu has the following options:

    • Authentication Tree
    • Generate Access Control File
    • Access Control Profiles
    • Access Control Groups

    To properly configure access control within Directory Manager:

      1. Add a tree that will contain the authentication information (the groups and users that will be granted special administrative access), as well as Initialize it.
      2. Add any other trees, as desired. They must be configured to act as slave servers for the authentication tree, replicated from the master server.
      3. Configure the authentication tree's LDAP server (in slapd.conf) to replicate the Base DN to these other (slave) servers.
      4. Define an ACL Profile, with the desired groups. Trees with more than one profile will permit access from ANY of the groups of any of the profiles.
      5. Generate the DMACL file (saving it as DMACL.conf), and redistribute it to each slave server, adding an 'include DMACL.conf' directive to slapd.conf and restarting.
      6. Define the members of these groups via the 'Click Here' link on the Access Control menu.
  10. Licensing Menu:

    To properly configure licensing within Directory Manager, you need to distribute your licenses among the one or more directory trees that you have. This allows you to allocate your license properly, to your liking. To purchase more licenses, please visit http://www.malysoft.com/products/

    The list at the top shows the number of licenses you have allocated. The next list shows how many licenses are yet to be allocated to a specific tree. Each tree is then displayed with the distribution. You can enter in the number you wish to allocate for a particular tree's module, or you may use '-' or '+'. You may also click on the 'Remainder' radio button for a module to distribute all remaining licenses to a particular tree.

    You have the option to automatically allocate licenses when an entry is added, even if the amount already allocated is used up, AS LONG AS you have licenses unallocated. Simply click on 'Automatically allocate new licenses upon Add' to enable this. However, you may want to disable this (the default) if you want to keep track of your license distribution more closely.
