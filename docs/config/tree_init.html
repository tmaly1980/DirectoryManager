Directory Manager 2.0 Documentation: Tree Initialization

If you have not initialized your directory (via Directory Manager) with the proper container entries (i.e. the 'ou=Users' container et al), then it must be populated with such. The password provided is for the Root DN as specified earlier in the configuration. The tree MUST be at least specified in the OpenLDAP server configuration for this to work! Running this will not delete anything pre-existing. If you want to use SSL, enable it first from the SSL/TLS menu.
