Directory Manager 2.0 Documentation: Mail Server Integration

Directory Manager implements Mailing Lists using the 'nisMailAlias' schema, under the 'ou=Aliases,BASEDN' subtree of your respective Directory Tree. The mailing list name is provided by the 'cn' attribute and the member email (whether local such as 'john' or absolute email such as 'john@doe.com') is provided by one or more 'rfc822MailMember' attributes within the entry.

Directory Manager ALSO makes it feasible to use your mail server to provide ALIASES to users via their user entries. This is done by performing filters based upon the 'inetOrgPerson' objectClass (i.e.: '(objectClass=inetOrgPerson)'), and searching for the 'mail' attribute (which can be local or absolute in the form of 'user@domain', in which you'd want to have the LDAP filter: '(&(objectClass=inetOrgPerson)(mail=*@DOMAIN))'.

For configuring your SMTP server for use with OpenLDAP and the Mailing List module as provided by Directory Manager, please consult the links below, depending on your SMTP server implementation. These links are provided by third parties, and is not provided by Maly Soft. Note that other mail servers MAY suppot LDAP, but merely are not linked below.
