What is Directory Manager?
Directory Manager is a powerful and extensible LDAP Directory Management Suite that enables you to manage directory entries of one or more OpenLDAP servers, including:
  • Address Book Entries (i.e. Rolodexes)
  • UNIX Users and Groups
  • Samba Users and Groups
  • SMTP Mailing Lists/Aliases
  • Room/Office Locations

Using Directory Manager together with OpenLDAP and Samba (as well as other third party server software), you can:

  • Centralize UNIX and Windows (Samba) users and groups for an entire network into a single database managed from one interface
  • Manage one or more distinct UNIX/Windows account databases from a single interface
  • Consolidate UNIX and Samba users and groups into a single data entry, including proper account/password synchronization
  • Replace expensive Windows-based domain controllers with inexpensive Linux-based Samba servers, while keeping a convenient management interface
  • Manage email mailing lists and aliases for one or more domains from a single interface
  • Migrate from using an /etc/passwd or NIS style user database to a centralized LDAP database that can serve entire networks
  • Migrate from a pure UNIX or Samba 2 environment to Samba 3 easily

Directory Manager provides unparalleled power and convenience through the following features:

  • Rapid form entry through customizable drop-down lists as well as pre-filled entry templates
  • Viewing and editing related cross-referenced entries from a single page
  • Embedding a searchable LDAP directory into your pre-existing web pages
  • Data importing for migrating external databases (such as /etc/passwd or NIS) into LDAP
  • Exporting LDAP searches to Excel-compatible CSV format
  • Management of multiple LDAP directories and servers from a single interface
  • User friendly browsing of entries in an address-book style
  • Managing cross-referenced (related) entries (such as a user's auxiliary groups or mailing lists) from a single page (such as the user's entry page itself)
  • Easily mix and match entry types (such as address book, UNIX or Samba users) within the same directory node on a per-entry basis
  • Perform systematic (bulk) changes to one or more entries at one time, such as disabling accounts or changing departments during a corporate 'reorganization'
  • Distribute several levels of administrative access via access control groups, incuding 'Full Administrator', 'Moderator' and 'Requestor' levels of access, separating common data entry and maintenance from internal systems control
  • Define access control groups once, and assign to one or more tree
  • Seamless migration to Samba 2 or Samba 3 integrated directories
  • Automatically create home directories for users via a secure, simple and extensible HTTP(S)-based RPC Daemon
Other major features of Directory Manager include:
  • Secure communications through TLS/SSL
  • LDAP-based access control
  • Multiple levels of administrative access
  • Access levels determined by group membership
  • Restricting module access on either a global or tree-by-tree basis, based upon various access level requirements
Why use Directory Manager?
  • Is standards compliant, using technologies such as LDAPv3, TLS, OpenLDAP access control lists, as well as standard LDAP schemas
  • Is written in Perl, a common interpretive programming language for UNIX/Linux that is hacker friendly (unlike compiled Java or C++ code)
  • Performs directory access and modifications as specific user entries (the logged-in user) instead of the Root DN for minimal impact on security
  • Helps promote using the customizability and stability of open source (OpenLDAP, Samba) while providing the power and convenience of a graphical frontend
  • Does not lock down nor prevent management of underlying technologies (LDAP, Samba, etc) through other third-party tools
Why is Directory Manager commercial?
Directory Manager is commercial (rather than open source) to provide:
  • Software support
  • Documentation
  • A user friendly interface
  • Convenient use of underlying open source software
  • Extensive customization while having sensible defaults
  • Full-time dedicated development and support, to help the project flourish rather than remain as a sporadic side project in the developers' free time

Directory Manager is priced according to the scale of it's use. We hope to provide Directory Manager to large-scale users at a fair price, while still enabling users with few and small directories to be able to use our software at an affordable price.

Software Prerequisites
  • Linux/UNIX based server
  • OpenLDAP 2.x Server
  • OpenSSL (if you wish to use TLS, recommended)
  • Apache 1.3+ web server software
  • Perl 5.6.1+, with recent versions of the following Perl modules (available from http://search.cpan.org/):
    • Net::LDAP
    • Digest::SHA1
    • Crypt::SmbHash
    • Net::SSL (uses OpenSSL)
    • HTTP::Daemon (non-SSL) and HTTP::Daemon::SSL (uses OpenSSL)
