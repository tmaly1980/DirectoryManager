Home Page

Once you open the URL for Directory Manager in your browser, you will reach a home page that will redirect you to the user interface. As the Systems Administrator configuring Directory Manager, you can do so by clicking on 'Administration' to go to the configuration tool.

One or more directories are managed through a single graphical interface. Usage can be provided both anonymously (for address-book style searching), as well as authenticated (for modifying entries, including your own corresponding user entry).

Logging In

Whether or not you are forced to log in (or can browse anonymously) depends upon configuration settings. If the Login page is not automatically displayed, you can click on 'Login' to then log in as a particular user with special rights.

The login page lets you log in either by a username or an LDAP DN (usually in the form of 'uid=user,ou=People,dc=domain,dc=com'). If logging in as a user, you must specify which tree the account is in. IF you log in using an LDAP DN, the 'Tree' drop-down list will be ignored.

You can use Directory Manager anonymously, without logging in, by selecting the desired tree and clicking on 'Anonymous Login'. This will only work if anonymous usage is enabled on the specific tree.

Once logged in (by clicking on 'Login', if not enforced), your name and access level (if applicable) will show up in the top-left portion of the screen. Clicking on your name will let you change your account/entry information (unless you log in as a Root DN, in which there is no entry within LDAP to modify):

A drop-down list provides switching between directory trees:

A tab-oriented menu provides switching between different types of entries. Which tabs are available depends on configuration settings, and is customizable per tree.

Currently, the following entry types are supported:

  • Users: UNIX users, Samba users, or standard address book users (can be mixed), including personal, employment and contact information (phone, email and postal address).
  • Groups: UNIX groups, Samba groups
  • Mailing Lists: SMTP mailing lists and aliases
  • Locations: Office/Room/Building information
  • Access Groups: Internal access lists to determine administrative access for Directory Manager

Within each tab, there are several navigational links, depending on your level of access, including:

  • Browse: List all entries
  • Search: Find specific entries (to view or edit) based upon given criteria
  • Requests: View the queue of pending entry requests
  • Import: Import a file containing entries from an external database (/etc/passwd, XLS, etc)
  • Add: Add new entries to the tree
Browsing Entries

By clicking on 'Browse', all entries in the tree for the given entry type will be displayed. Large directories will take a while to load. You can sort based upon a particular column by clicking on the field name.

Searching Entries

By clicking on the 'Search' link, you will be presented with a search form.

This form allows you to specify a specific attribute within the entry to match. It also allows you to perform approximate matching by providing a drop-down list specifying how to match the attribute (i.e. 'contains', 'is', 'starts with', 'ends with'). You can then specify how many results to show per page, or 'All' for no paging.

You can click on 'Search' to go to the results page, or click on "I'm Feeling Lucky" to AUTOMATICALLY view (the details of) the first entry found (for example, if you know that there is only one entry given the criteria you enter).

Search Results

Search results display a short summary of the entries, giving commonly sought information. More details are available by going to the 'view' page for an entry, by clicking on the entry's name.

Search results are 'paged', meaning that only a certain number of entries display per page. Depending on configuration settings, paging can occur in two ways:

Results can be divided alphabetically, with a single page per letter. There is an alphabetical navigation at the top of the page to switch between letters, or to specify 'All':

Advanced Search
 Within the Search screen is a link to perform an advanced search. Clicking on 'Advanced Search' presents you with a larger menu of options. Clicking back on 'Simple Search' takes you to the original page.

From the advanced search page, you can specify searching for records based upon a list of possible matches for a given field. For example, searching for all entries whose username is either 'jdoe', 'mstevens', or 'wrichards'. Simply click on 'Add'/'Remove' to alter this list.

You can also enter in an LDAP filter (RFC 2254) to search by. Clicking on the 'LDAP Filter' link will take you to the online reference.

Finally, you can export the results into an Excel CSV file for external use, by clicking on 'Export to Excel/CSV'.

Viewing Entries

By clicking on the name for an entry from the Browse or Search pages (assuming the 'Implicit Edit' configuration option is disabled), you are presented with a more detailed record of an entry. There will also be a navigational link to switch to 'Edit' mode (and back) of an entry if you have such access.

Items are subdivided within sections, which can be shown or hidden (for viewability) by clicking on the corresponding checkbox.

The information provided is based upon one's level of access. As an administrator, you see relevent internal information for systems use that is otherwise irrelevent for common users (i.e. UNIX/Samba account information). As a common user, you only see information relevent to you, such as a person's contact information.

Entry Sections

Entries contain data in relevent sections such as 'Contact Information' and 'UNIX Account'. Each section contains a checkbox next to the name that allows for quickly hiding the respective content. This allows you to get rid of the data you don't want to see, and see the data you want.

Cross-referenced entries

Depending on system configuration, an entry may provide inline displaying of related entries (such as the auxiliary groups of a user). If this is enabled (which has performance impacts), viewing an entry will show these cross-referenced entries. You can THEN click on a cross-referenced entry to view IT in a separate window.

If not enabled (which provides better performance), then you will simply be provided with a 'View' link to open up a search results page of such related entries.
